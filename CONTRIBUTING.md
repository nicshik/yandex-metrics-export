# Contributing

Thanks for improving Yandex Metrics Export.

## Development

1. Install dependencies with `npm install`.
2. Start the local app with `npm run dev`.
3. Run `npm run check` before opening a pull request.

## Pull Requests

- Keep changes focused and describe the user-facing impact.
- Do not commit OAuth tokens, counter IDs, exports, screenshots with private data, or analytics reports.
- Keep examples generic. Use placeholders such as `12345678` or `<COUNTER_ID>`.
- Update README or in-app guidance when behavior changes.

## Reporting Issues

When filing a bug, include:

- the steps to reproduce;
- the expected and actual result;
- relevant browser and Node.js versions;
- sanitized request details without tokens or real counter IDs.
